# О проекте

Репозиторий отражает практическую модель внедрения AppSec в SDLC.

В рамках работы над реальными кейсами я:

- систематизирую уязвимости по языкам, фреймворкам и инфраструктуре  
- формализую паттерны эксплуатации и remediation  
- разрабатываю правила и конфигурации DevSecOps-инструментов для автоматического обнаружения  
- внедряю security-gates в CI/CD  
- снижаю риск попадания уязвимого кода в production  

**Цель проекта** — не каталог уязвимостей, а воспроизводимый security-процесс:  
от выявления риска до автоматизированного контроля и предотвращения регрессии.

---

# Подход

Репозиторий наполняется по воспроизводимой методологии:

1. Определяю attack surface языка и выбранного контекста
2. Формирую общий пул high/critical уязвимостей
3. Мапплю уязвимости на конкретные фреймворки
4. Реализую уязвимый код
5. Демонстрирую эксплуатацию
6. Анализирую первопричину
7. Реализую исправление
8. Добавляю правила детектирования (DevSecOps инструменты)
9. Интегрирую кейс в общий security-pipeline
10. Внедряю автоматизированную проверку регрессии

# Наполнение

Каждый кейс включает:

- описание класса уязвимости и threat-модели  
- пример эксплуатации  
- анализ первопричины  
- исправление  
- правила для SAST / DAST / dependency scanning  
- конфигурацию CI-проверок  
- стратегию предотвращения повторного появления  

Репозиторий демонстрирует практический подход к:

- построению AppSec-контролей  
- автоматизации security-проверок  
- снижению количества уязвимостей до релиза  
- интеграции security в процесс разработки без деградации velocity  

Это рабочая модель снижения риска через автоматизацию и процессный контроль.

# Навигация

От корня репозитория:

- `languages/` — примеры уязвимостей по языкам и направлениям: **python**, **javascript**, **typescript**, **java**, **dotnet**, **c**, **cpp**, **erlang**, **go**, **rust**
  - Внутри каждого языка контекст: `web/`, `cli/`, `system/`, `mobile/`, `embedded/`
  - Каждый кейс содержит:
    - `vuln_app.*` — уязвимый код  
    - `fixed_app.*` — исправленная версия  
    - `exploit_payload.txt` — демонстрация атаки (если применимо)  
    - `README.md` — описание кейса

- `infrastructure/` — примеры уязвимостей и best-practice по Cloud, Orchestration, Containerization, IaC

- `devsecops/` — конфиги и примеры инструментов безопасности: **sast**, **dast**, **fuzzing**, **secrets**, **sca**, **iac-scanning**, **ci-cd-pipelines**
